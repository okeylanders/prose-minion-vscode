Model Browser — replaces the model dropdown

The dropdown could only show a name. This is a full browse-and-pick surface built in the same warm-brown language as the Writing Tools picker — but sized for the narrow sidebar column, so it drops in wherever the old Claude Opus 4.8 ▾ control lived. Tall and scrollable; each model carries its live price, context window, family, release date, and a full description.

It's interactive: toggle By Provider / By Family, filter with the chip row, search, and click a card to select it. Coral (not the generic VS Code blue) keeps it native to PM. Pricing & counts mimic a live OpenRouter pull; the last card shows the offline/fallback state.

Anatomy & decisions
  • Header — eyebrow → bold title → one-line subtitle, with a ghost close (✕) that returns you to the view, mirroring the Tools picker header exactly.
  • Search + two pivots — By Provider and By Family re-group the list and swap the chip row. A leading All (82) chip is selected by default.
  • Price pill — $in / $out with a per 1M caption, right-aligned and tabular so columns of numbers stay legible.
  • Badges — family, release date, and an accented ctx badge so the context window reads at a glance.
  • Full descriptions — 2–3 lines, never truncated. This is where the browser beats the dropdown.
  • Selected state — coral border, soft tint, and a corner check. Hover gives a quiet lift.
  • Offline case — Kimi K2.5 shows a dashed price pill with no numbers when OpenRouter rates can't be fetched.
