Prose Minion · Design refresh

Adopting the Frame Minion design language

I reskinned Prose Minion's chrome and all four sidebar tabs to match the warmer, calmer Frame Minion system — soft browns, a coral accent line under the header, icon-above-label tab cards, and a real balance widget top-right (OpenRouter, with the last-request cost beneath it). Result panels keep their monospace + emoji character; everything around them is now clean sans.

Then the bigger bet: the Assistant as a full VS Code editor tab — room to keep a conversation going, with contextual one-tap actions that change with the analysis. I drafted three directions so we can pick a lane.

These are static hi-fi comps — pixel-level look & feel, not yet wired. Once you approve a direction I'll build it into a fully clickable prototype with mock streaming. Assumptions: PM uses OpenRouter only (no FAL.ai), so the widget shows one balance + last-request cost. Tool icons are Lucide-style placeholders; swap for your real set anytime.

Full-tab Assistant — three directions

Opened as a real editor tab. Keep chatting; act on the work. Pick a lane and I'll make it clickable.

Direction A — Conversation threadclassic chat

The familiar one. Excerpt and analysis live in the thread; a contextual action bar sits above the composer and rewrites itself per analysis type. Lowest-friction, most recognizable.

Direction B — Split & pinnedexcerpt stays put

The excerpt, context, and a tool palette pin to a left rail so they never scroll out of reach; the conversation runs on the right. Best when you're iterating tightly on one passage.

Direction C — Branch boardthe wild one

Every analysis and rewrite is a card on a spine. Branch any card to explore variations in parallel — nothing touches your draft until you hit Apply. Non-linear, great for "show me options" energy.
